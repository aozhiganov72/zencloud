# zencloud
#### инструменты для работы с облаками
Для работы скриптов необходима предварительная установка zenity, rsync, sqlite3 а так же установка и настройка rclone.
Инструкции по настройке rclone наверняка можно найти в интернете. Позже возможно дам ссылку.
Использовать скрипты можно скопировав их в любую папку на выбор пользователя. Туда же необходимо положить и файл базы данных cloud.db
Затем необходимо создать (например на рабочем столе) кноки запуска следующих скриптов: 
1. Manager.sh
2. Cloud.sh
3. Unmont.sh

 ### Важно
Не забудте сделать все скрипты исполняемыми.

Скрипт Cloud.sh создаст в домашней папке пользователя папку Clouds в которой будут созданы папки для монтирования настроеных в rclone облаков.
А так же будет их примонтировать. Автоматическое монтирование при старте системы можно настроить средствами вашей системы.
Размонтирование осуществляется скриптом Unmont.sh. 

Скрипт Manager.sh используется для выбора действия:
1. Копирование одиночного файла в облако
2. Копирование папки в облако
3. Копирование нескольких файлов в облако
4. Синхронизация папок (облако и локальная машина)

При этом синхронизация осуществляется шестью разными режимами на выбор пользователя.
Любое действие осуществляется с любым из ранее настроеных в rclone облаков, так же на выбор пользователя.
Скрипты адекватно работают с self-hosted webdav сервером на основе nginx.

Работоспособность с различными облаками зависит от возможностей rclone установленной в вашей системе.

## Предупреждение
Автор не несет ответственности за утрату вами файлов в случае некорректного использования скриптов пользователем.
Предварительно, особенно в режиме синхронизации, рекомендуется потренироваться в использовании на специально созданных для тренировки файлов и папок.
